資訊安全與攻防：簡要規則集 ​

資訊安全（Information Security）是保護資訊系統免受未經授權的存取、使用、洩露、破壞、修改或銷毀的實踐。本規則集旨在提供資訊安全領域的核心概念、常用工具與防禦策略的快速指南。

雖然技術日新月異，但安全的核心原則始終如一。無論是紅隊（攻擊方）還是藍隊（防禦方），都必須遵循法律與道德規範。

核心原則 ​

  1. CIA 三要素（Confidentiality, Integrity, Availability）：這是資訊安全的基石。機密性確保只有授權人員能存取數據；完整性確保數據未被篡改；可用性確保授權用戶在需要時能存取資源。
  2. 最小權限原則（Principle of Least Privilege）：用戶或程式應僅擁有完成其工作所需的最小權限。這能限制攻擊者在系統遭入侵時的橫向移動能力。
  3. 縱深防禦（Defense in Depth）：不要依賴單一的安全措施。應採用多層次的防禦機制（如防火牆、入侵檢測系統、端點保護），若一層防禦失效，其他層仍能提供保護。
  4. 零信任架構（Zero Trust）：永遠不要信任，始終驗證。無論請求來自內部網路還是外部，都必須經過身份驗證和授權。

攻防工具與實作 ​

安全從業人員常使用各種工具進行滲透測試或漏洞評估。使用這些工具時，請務必在授權環境下進行。

  1. 網路掃描與偵查：

    • Nmap：網路探索與安全審計的標準工具，用於發現主機、服務及其版本。
    • Wireshark：網路封包分析軟體，用於截取並詳細檢查網路流量。

  2. 密碼破解與認證測試：

    • John the Ripper：快速的密碼破解工具，支援多種雜湊演算法，常用於檢測弱密碼。
    • Hashcat：基於 GPU 的進階密碼恢復工具。

  3. 無線網路安全：

    • Aircrack-ng：一套用於評估 WiFi 網路安全的工具，包含監控、攻擊、破解和測試 WiFi 卡驅動程式的功能。

  4. 漏洞利用與滲透測試框架：

    • Metasploit Framework：用於開發和執行漏洞利用程式碼的強大平台，是滲透測試的必備工具。
    • Burp Suite：用於 Web 應用程式安全測試的整合平台，包含代理伺服器、掃描器和入侵工具。

防禦與最佳實踐 ​

  1. 定期更新與修補：軟體漏洞是攻擊者的主要目標。保持作業系統、應用程式和韌體的最新狀態是防禦的基礎。
  2. 強身份驗證：強制實施多因素驗證（MFA），並要求使用強密碼或密碼管理器。
  3. 加密通訊：使用 SSH、TLS/SSL 等協議加密傳輸中的數據，並對靜態數據進行加密儲存。
  4. 備份與災難復原：定期備份關鍵數據並測試復原流程，以應對勒索軟體攻擊或系統故障。
  5. 安全開發生命週期（SDLC）：在軟體開發的早期階段就納入安全考量，進行代碼審查和靜態/動態分析。
